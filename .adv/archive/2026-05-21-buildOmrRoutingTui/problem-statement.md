OMR currently depends on an imported/legacy OMP-shaped UI for configuring model preferences, but model routing now needs a first-class tool built around OMR’s own concepts: primary model, fallback chain, routing policy, validation, preview, and plugin readiness. The desired outcome is an OMR-native successor that reuses only the useful OMP pieces while making OMR self-contained and not dependent on the old `omp` project or UX model.
